Основной акцент смещён на отделение структуры данных от поведения через интерфейсные соглашения, а также на использование функций высшего порядка как базового инструмента обработки коллекций.

---

### Обобщённая типизированная коллекция (TypedCollection)

Была реализована универсальная коллекция `TypedCollection[T]`, основанная на использовании обобщённых типов (`Generic` и `TypeVar`).

Коллекция поддерживает хранение элементов любого типа и предоставляет базовый набор операций:

- добавление и удаление элементов;
- доступ по индексу;
- получение копии всех элементов;
- итерация по коллекции.

Данный подход обеспечивает переиспользуемость структуры без привязки к конкретной предметной области.

---

### Функции высшего порядка

Коллекция реализует функциональный интерфейс для обработки данных, включающий:

- `filter(predicate)` — фильтрация элементов по условию;
- `find(predicate)` — поиск первого подходящего элемента;
- `map(transform)` — преобразование элементов в новый тип;
- `apply(func)` — применение функции ко всем элементам;
- `sort_by(key)` — сортировка по вычисляемому ключу.

Такой подход позволяет работать с данными в декларативном стиле, минимизируя необходимость явных циклов и повышая читаемость кода.

---

### Протоколы и структурный полиморфизм

В рамках работы были использованы протоколы `Protocol`, задающие поведенческие контракты:

- `ComfortIndex` — определяет возможность вычисления индекса комфортности;
- `RentalFeasibilityIndex` — определяет возможность оценки арендной привлекательности.

Использование `@runtime_checkable` позволяет проверять соответствие объектов протоколам во время выполнения.

Данный подход заменяет классическое наследование и реализует структурный полиморфизм (duck typing с формальной типизацией), где важен не класс объекта, а наличие необходимого поведения.

---

### Доменные модели и разделение ответственности

В системе используются различные доменные наборы данных:

- приватные объекты недвижимости (расчёт комфортности);
- коммерческие объекты недвижимости (расчёт арендной привлекательности).

Каждый набор данных обрабатывается через единую типизированную коллекцию, но с разными поведенческими контрактами, что демонстрирует гибкость архитектуры.

---

### Преобразование данных (map и проекции типов)

Особое внимание уделено преобразованию данных между типами:

- объекты → строки (адреса);
- объекты → числовые значения (стоимость, индексы);
- объекты → вычисляемые характеристики (`comfort_index`, `rental_feasibility_index`).

Это демонстрирует возможность использования коллекции как слоя абстракции над доменной моделью.
