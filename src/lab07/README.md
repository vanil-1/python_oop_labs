# Лабораторная работа №7 — Консольное приложение (Python 3.x)

## Цель работы

* Объединить все знания, полученные в ЛР1–ЛР6, в единое работающее приложение;
* Реализовать интерактивный CLI-интерфейс с меню и вводом пользователя.

---

## Ход работы

В ходе выполнения лабораторной работы была разработана полноценная консольная утилита для управления районом недвижимости (HousesDistrict). Приложение построено на основе ранее созданных классов домов (PrivateHouse, CommercialHouse), коллекции (HousesDistrict), стратегий сортировки и фильтрации из ЛР5, а также дополнено трёхслойной архитектурой, обработкой ошибок, сериализацией в JSON и интерактивным меню с цветным выводом.

---

### Ключевые решения:

Разделение на слои:

* models.py (импортируется из src.libs.models) — классы предметной области;

* app.py — бизнес-логика, управление коллекцией, загрузка/сохранение;

* cli.py — только ввод/вывод, вызов методов app;

* storage.py — сериализация/десериализация в JSON;

* exceptions.py — собственные исключения.

Интерактивное меню: 9 пунктов (добавление, вывод всех, поиск, фильтрация, сортировка, удаление, заключение договора, сброс характеристик дома, выход).

Обработка ошибок: перехват ValueError, собственных исключений (InvalidMenuChoiceError, HouseNotFoundError, StorageError и др.).

Сохранение/загрузка: автоматическая загрузка при старте, сохранение при выходе (с подтверждением).

Типизация и docstring: все публичные методы аннотированы и документированы.

---

### Назначение модулей:

Файл - ответственность:

* main.py - создание экземпляра HouseApp и CLI, запуск цикла;

* cli.py - отрисовка меню, баннера, статистики, таблиц. Ввод пользователя, вызов методов app, обработка исключений, анимация загрузки;

* app.py - бизнес-логика: добавление/удаление домов, поиск, фильтрация, сортировка, аренда/сброс, загрузка/сохранение через storage;

* storage.py - преобразование объектов House в dict и обратно, чтение/запись JSON-файла;

* exceptions.py - иерархия исключений приложения;

* color_map.py - константы цветов для форматированного вывода в терминале.

---

### Описание CLI

При запуске python main.py отображается:

Баннер с названием, версией, разработчиком.

Список выполненных лабораторных работ.

Статистика района: общее количество домов, частных/коммерческих, в аренде/свободных.

Меню с 9 пунктами.

---

### Пункты меню

Add house — добавление нового дома (частного или коммерческого). Запрашиваются все необходимые поля.

Show all houses — вывод всех домов в форматированной таблице.

Find house — поиск дома по полю (адрес, этажи, стоимость, площадь).

Filter houses — фильтрация по условиям: не в аренде, стоимость > X, площадь > X.

Sort houses — сортировка по адресу, стоимости, площади или этажам.

Remove house — удаление дома по адресу с подтверждением (y/n).

Make contract — заключение договора аренды (установить rented = True).

Reset house — сброс статуса аренды (rented = False).

Exit — выход с предложением сохранить изменения.

---

### Обработка ошибок
При вводе нечислового значения в поле, ожидающем число, выбрасывается ValueError, перехватывается и выводится сообщение: "Invalid input type".

При выборе несуществующего пункта меню — InvalidMenuChoiceError.

При попытке удалить/найти/арендовать несуществующий дом — HouseNotFoundError.

Ошибки ввода типов дома, полей и т.д. обрабатываются и выводятся в красном цвете.

---

### Сохранение и загрузка
При запуске приложение автоматически загружает данные из data/lab07/data.json (если файл существует). Если файла нет, начинается с пустой коллекции.

При выборе 0. Exit запрашивается: "Save changes? (y/n)". При подтверждении данные сохраняются в JSON. При отказе — изменения теряются.

Формат JSON соответствует структуре домов: идентификатор типа (private/commercial) и все атрибуты.
