## Ход работы:

Так как предметная область согласно распределению - недвижимость, то было принято решение реализовать коллекцию HouseDistrict - район домов, а за основу модели для сущности класс House из прошлой лабораторной работы.

Класс `House` реализует сущность недвижимости с инкапсулированными полями и валидацией данных при создании объекта.

Основные особенности:
* закрытые атрибуты (_address, _cost и др.);
* доступ через @property;
* валидация входных данных через отдельный модуль;
* бизнес-логика:
  * заключение договора аренды;
  * расчёт стоимости аренды за период.

Магические методы:
* __str__() — человекочитаемое представление;
* __repr__() — техническое представление объекта;
* __eq__() — сравнение объектов по площади и стоимости.

Аналогично созданию модели, в коллекции также представлен конструкт __inti__() с объявлением 2-х полей: список сущностей House и имя(название) коллекции(района).

Согласно стандарту, в коллекции реализованны следующие функции:

## Управление данными

### add(self, other)

Метод добавления сущности в коллекцию, с валидацией сущности, причём метод не позволяет добавить дубликат. Дубликатом считается сущность с одинаковым адресом, так как если адреса совпадают, то существует 2 сценария. Первый - одинаковые остальные поля, тогда это абсолютный дубликат. Второй - разные остальные поля, тогда лучше обновлять поля через методы сущности, а не коллекции. Стоит обратить внимание и на то, что большая часть полей сущности не измены, кроме, например, стоимости, а их изменение - метод сущности, как и упомянуто ранее.

### remove(self, item)

Метод, позволяющий удалить сущность из коллекции. Аналогично прошлому методу, здесь также проходит валидация удаляющиеся сущности. Удаление возможно только в то случае, если сущность присутствует в коллекции.

### remove_at(self, index)

Аналогично remove(), но удаление происходит не по сущности, а по index. Также валидируется и значение index, так как не возможно удалить сущность по index, елси index выходит за пределы коллекции.

### get_all(self)

Позволяет получить сущности из коллекции.

### __getitem__(self, index)

Позволяет получить сущность по index, при этом также валидируется index, как и в remove_at()

## Поиск и фильтрация

### get_not_rented(self)

Позволяет получить список сущностей, которые на заняты(not rented). Если таковых нет, возвращает пустой список.

### sort_by_cost(self)

Сортирует коллекцию по стоимости сущностей(cost).

### find_by(self, field, value)

Этот метод реализует поиск сущностей по определённому атрибуту(field) и значению атрибута(value). Причём если коллекция пуста, то возвращает пустой список. Также наименование атрибута проверяется через mapping, расположенный в libs\config, в случае совпадения, метод ищет соответсвующий полю validator для валидирования значения атрибута. Метод возвращает список, если нет совпадений - список пустой, иначе - список с сущностями, соответсвующими значению поиска.

## Операции с коллекцией

### __len__(), __iter__(), __str__(), __repr__()

"Магические методы". __len__() - длина коллекции(количество элементов/сущностей), __iter__() - позволяет проходить по коллекции, как по списку, __str__() - выводит коллекцию в читаемом(табличном) виде (print(collection)), __repr__() - выводит коллекцию в удобном для разработчика виде.
