## Базовый класс House

Класс `House` представляет собой абстрактную модель недвижимости.

Основные особенности:

* использование абстрактного класса (ABC);
* инкапсуляция данных;
* валидация входных данных через отдельные модули;
* базовая бизнес-логика:
  * заключение договора аренды;
  * расчёт стоимости аренды за период;
* определение базового контракта через абстрактный метод:
  * `value_efficiency_index()`.

---

### Базовый контракт

В рамках лабораторной работы введён единый контракт value_efficiency_index(). Данный метод определяет эффективность объекта недвижимости.

Особенности контракта:

* одинаковое назначение для всех наследников;
* различная реализация в дочерних классах;
* используется для демонстрации полиморфизма.

---

## Иерархия классов

На основе базового класса `House` реализованы производные классы:
House
├── PrivateHouse
└── CommercialHouse

---

## PrivateHouse

Класс `PrivateHouse` представляет частный жилой дом.

### Дополнительные атрибуты:
* land_area — площадь участка;
* heating_type — тип отопления;
* occupants_count — количество жильцов.

### Особенности:
* расчёт комфортности жилья;
* переопределение метода:
  * `value_efficiency_index()` → `comfort_index()`;
* использование весовой модели оценки.

---

## CommercialHouse

Класс `CommercialHouse` представляет коммерческую недвижимость.

### Дополнительные атрибуты:
* usage_type — тип использования (office, retail, warehouse, hotel);
* operational_area — полезная площадь;
* customers_average_count — среднее количество клиентов.

### Особенности:
* расчёт эффективности аренды;
* переопределение метода:
  * `value_efficiency_index()` → `rental_feasibility_index()`;
* учёт типа бизнеса и нагрузки на объект.

---

## Полиморфизм

В рамках работы реализован полиморфизм:

* одинаковый метод вызывается у разных классов;
* поведение метода зависит от типа объекта;
* отсутствует необходимость в проверках типа (`if type == ...`).

Пример: house.value_efficiency_index().

---

## Коллекция объектов (HousesDistrict)

Для работы с объектами использован класс коллекции `HousesDistrict` из ЛР-2. Коллекция поддерживает:

* хранение объектов разных типов (PrivateHouse, CommercialHouse);
* итерацию по объектам;
* фильтрацию по типу;
* поиск и сортировку;
* защиту от дубликатов.

---

## Дополнительно: загрузка данных (JSON)

Для дочерних классов реализован дополнительный модуль загрузки данных из файлов .JSON - convert_pr_cl_to_list() который:

* читает JSON;
* определяет тип объекта;
* создаёт соответствующий класс;
* возвращает список объектов.
